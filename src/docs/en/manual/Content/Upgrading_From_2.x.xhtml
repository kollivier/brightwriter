Upgrading From 2.x

If you've been working with previous versions of BrightWriter, you will find that version 3.0 is a major change from previous versions. These changes were necessary to keep BrightWriter relevant as a tool for building modern learning modules and e-books that work with the commonly available software out there today.

To get the most out of BrightWriter 3.0, you should consider changing the way you work with the program. For example, typically in the past, one would create content in a separate HTML editor and then import those contents into BrightWriter. While this is still possible, you will get better results by using the built-in HTML editing functionality, and if you do decide to import, it is highly recommended you run the HTML Clean Up tool available from Format -> Clean Up HTML option in the menu bar.

Also, another significant change to be aware of is that the project file automatically saves itself whenever you make a change. You no longer need to save projects. However, you do need to save changes to pages as you edit them. So the Save functionality still exists, but it applies to pages now, not the project itself.

Lastly, a couple features from previous versions have been removed, some temporarily and some permanently. The main feature that has been removed permanently is the functionality for setting themes. Very few users created their own themes, and these themes would not work well with the new editing mode, nor would it be properly supported when the EClass was exported as a learning module
or e-book. Therefore the decision was made to discontinue this feature, as it was felt that the ability to support Learning Management Systems (LMS) and e-book readers was a core functionality for the program. Also, search functionality in exported EClasses has been removed as well. It required a lot of setup on the server side, which was sometimes highly technical and problematic. If you export as a learning module or e-book, chances are the device used to read the EClass has built-in search functionality, so the search functionality has more been replaced by better solutions than removed.

The main features that have been temporarily removed are the functions for embedding videos and creating tests. These are both related to issues with exporting to various places. Once the technical issues with this have been resolved, these capabilities will be added back into the program. We appreciate your patience with this.

Overall, the changes in BrightWriter resolve a lot of long-standing issues with the program, such as the problematic hotword support (where sometimes it would not properly detect and link hotwords) and issues supporting non-English EClasses often caused by HTML programs which would not do things such as identify the language the HTML page was written in. All pages you create now conform strictly to the XHTML 1.1 standard, which should reduce problems with viewing among browsers and allow for even more interesting additions in the future.

Thank you for your support for the program, and hopefully this will help you to create even better EClasses!
